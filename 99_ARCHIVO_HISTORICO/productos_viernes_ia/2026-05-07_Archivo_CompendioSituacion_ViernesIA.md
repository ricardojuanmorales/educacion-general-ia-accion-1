# Compendio de Situación
## Viernes de IA 2026S21
### Punto de partida para implementación desde Producto 07 - Análisis de Aplicabilidad Framework V4.1.0

**Proyecto:** Inteligencia Artificial Integrada y Educación General: Viernes de IA 2026S21  
**Estado:** N3 consolidado con proyección N4  
**Propósito inmediato:** comenzar implementación institucional usando el Producto 07 como arquitectura interna de continuidad.

---

## 1. Situación actual

El proyecto ha evolucionado desde un calendario de talleres sobre inteligencia artificial hacia una arquitectura formativa con potencial institucional. La versión pública del calendario ya integra ciclos, títulos macro con respiración tipográfica, descripciones convocantes, productos por sesión, ética, accesibilidad y una sección final de proyectos transversales.

El Producto 07 permite pasar de la fase de diseño a la fase de implementación. Su función es organizar el calendario como proyecto evolutivo permanente mediante niveles de madurez, perfiles, Caleidoscopio, gates, CHECKS, gobernanza documental, evidencias y ruta de institucionalización.

---

## 2. Documentos producidos y función de cada uno

### Producto 03 - Bosquejo de Integración Macro-Calendario
Sirvió como puente entre el calendario vigente, la propuesta macro y las recomendaciones diagnósticas. Definió la fórmula común:

**Pregunta educativa → práctica situada → producto → evidencia → reflexión ética → continuidad**

### Producto 04 - Matriz de Alineación e Integración del Calendario
Es la sala de máquinas académico-operativa. Integra doble titulación: título académico completo y título público con respiración tipográfica. Incluye preguntas educativas, productos, evidencias, ética, accesibilidad, continuidad, riesgos y mitigaciones por sesión.

### Producto 05 - Calendario Público Revisado y Actualizado
Es la versión pública lista para circulación. Integra frase marco, ciclos, sesiones, objetivos, productos, práctica ética-accesible y la sección de cinco proyectos transversales.

### Producto 06 - Sección de Proyectos Transversales
Analiza el estado del Producto 05 e integra los cinco proyectos de continuidad como evolución orgánica del calendario:
1. Ambiente Virtual de Aprendizaje Transdisciplinario.
2. Laboratorio Vivo para el Estudio Humano-Máquina.
3. Serie de Podcasts sobre Educación General e Inteligencia Artificial.
4. Certificación Educativa Global vía MOOC.
5. Investigación Educativa Sistémica, Holística y Transdisciplinaria - Framework V4.1.0.

### Producto 07 - Análisis de Aplicabilidad Framework V4.1.0
Documento rector para la implementación. Incluye:
- diagnóstico de necesidad;
- nivel N3/N4;
- activación de perfiles;
- activación Caleidoscopio;
- gates y CHECKS;
- documentos mínimos;
- gobernanza documental;
- riesgos;
- ética y accesibilidad;
- evidencias de avance;
- continuidad y archivo;
- ruta de institucionalización.

### Anexo A - Guía Académico-Operativa para Facilitadores
Documento de ejecución para quienes faciliten los talleres. Define estructura de dos horas, roles, preparación, fichas operativas, documentación mínima y checklist ético-accesible.

### Anexo B - Kit de Plantillas del Participante
Incluye plantillas para mini-bitácora, producto con IA, prompt académico, ficha ética-accesible, recursos textuales, audio, imagen, video, prototipo, investigación, sistema académico, curaduría, evaluación y mini-simposio.

## 3. Decisiones tomadas

1. El calendario público conservará tono promocional, humano, práctico, universitario, colaborativo y convocante.
2. Los títulos de la propuesta macro serán la identidad académica principal.
3. Se usará doble titulación:
   - título académico completo;
   - título público con respiración tipográfica.
4. El modelo educativo se mantendrá implícito en la versión pública.
5. Ética y accesibilidad se integrarán como prácticas normales, no como apéndices.
6. Los proyectos transversales deben parecer evolución orgánica del calendario.
7. El Producto 07 será el documento rector para iniciar implementación.
8. El estado actual es N3 consolidado con proyección N4.
9. El Framework V4.1.0 funcionará como arquitectura interna de institucionalización.

---

## 4. Diagnóstico actual

### Fortalezas

- El calendario ya tiene una arquitectura de ciclos clara.
- La versión pública es convocante y conserva profundidad conceptual sin sobrecargar.
- Cada sesión tiene producto tangible.
- Ética y accesibilidad aparecen como práctica transversal.
- Existen proyectos transversales que abren continuidad institucional.
- El Producto 07 permite pasar de diseño a implementación.

### Tensiones

- Falta contrastar el Producto 07 con el documento completo del Framework V4.1.0.
- Falta formalizar repositorio documental.
- Falta definir responsables por ciclo, evidencia, archivo y proyectos transversales.
- Falta convertir la versión actualizada del Producto 05 en documento institucional final.
- Falta operacionalizar gates, CHECKS y gobernanza en una ruta de trabajo fechada.

### Riesgos principales

1. Sobrecarga documental.
2. Fragmentación entre talleres.
3. Evidencia dispersa.
4. Ética tratada como discurso y no como práctica.
5. Accesibilidad incorporada tarde.
6. Falta de responsables de archivo.
7. Intentar escalar a MOOC antes de validar el piloto.
8. Dependencia de pocas personas.
