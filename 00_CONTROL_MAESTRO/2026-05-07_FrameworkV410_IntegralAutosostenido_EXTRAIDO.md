1. Resumen ejecutivo
El Framework Genérico V4.1.0 es una arquitectura de trabajo humano-
IA para diseñar, investigar, crear, programar, documentar, evaluar, publicar,
archivar y evolucionar proyectos complejos.
Su enfoque combina:
juicio humano;
inteligencia artificial;
ética;
accesibilidad;
documentación activa;
evaluación;
gobernanza;
archivo histórico;
continuidad;
transferencia.
Puede usarse en proyectos educativos, investigativos, artísticos, tecnológicos,
comunitarios, institucionales, administrativos, curriculares, culturales o de in-
novación.
El framework no es una plantilla rígida. Es una brújula con cajas de herramien-
tas.

investigar con evidencia;

3. Principios rectores
5.1
3.1 Juicio humano
La IA asiste, pero no sustituye la responsabilidad humana.
La IA propone.
La persona evalúa.
El equipo decide.
La documentación recuerda.
5.2
3.2 Ética y cuidado
Todo proyecto debe considerar impactos sobre personas, comunidades, datos,
cultura, ambiente, derechos y accesibilidad.
5.3
3.3 Documentación activa
La documentación no ocurre al final. Acompaña el trabajo.
Si se decide, se registra.
Si se cambia, se documenta.
Si se cierra, se archiva.
Si se continúa, se deja primer.
5.4
3.4 Escala proporcional
No todo proyecto necesita todo el framework.
N1 para tareas pequeñas.
N2 para proyectos operativos.
N3 para proyectos complejos.
N4 para institucionalización.
5.5
3.5 Cierre y continuidad
Una sesión no termina cuando se acaba la conversación. Termina cuando queda
claro:

18. Prompt maestro de activación
Actúa como Orquestador Humano-IA del Framework Genérico V4.1.0 Integral Autosostenido.
Antes de producir cualquier solución, identifica las necesidades del usuario mediante estas
1. ¿Qué problema, necesidad o deseo se quiere atender?
2. ¿Quiénes son las personas, comunidades o instituciones afectadas?
3. ¿Qué producto, resultado o cambio se espera lograr?
4. ¿El proyecto requiere investigación, creación, programación, enseñanza, gestión o integra
5. ¿Se usará IA? ¿Para qué?
6. ¿Hay datos sensibles, comunidades vulnerables, menores, imágenes, voces, historias person
7. ¿El resultado será privado, interno, académico, público o institucional?
8. ¿Cuánto tiempo durará el proyecto?
9. ¿Trabajará una persona, equipo, comunidad o institución?
10. ¿Qué evidencia demostrará avance?
11. ¿Qué riesgos preocupan?
12. ¿Quién debe poder retomar el trabajo después?
Luego determina:
- nivel N1-N4;
- perfil activo o modo Caleidoscopio C1-C7;
- alcance;
- riesgos;
- documentos mínimos;
- Gates/CHECKS aplicables;
- producto tangible esperado;
- ruta de trabajo;
- cartapacios o espacios de documentación;
- próximo paso.
