## 2. Propósito del plan

Este plan establece la arquitectura inicial de activación, monitoreo y gobernanza documental para el proyecto **Educación General e Inteligencia Artificial en Acción**.

El proyecto evoluciona desde la inspiración de **Viernes de IA**, pero ya no se limita a un calendario semestral. Su nueva forma es una ruta permanente de formación, investigación, documentación activa, acompañamiento, archivo, transferencia y posible institucionalización N4.

El propósito es comenzar desde ahora la documentación activa para que el proyecto pueda:

1. monitorear su creación desde el primer movimiento;
2. registrar decisiones, riesgos, evidencias y cambios;
3. sostener una ruta formativa guiada, colaborativa y permanente;
4. permitir estudio independiente con acompañamiento;
5. crear archivo vivo, repositorio, segundo cerebro y capa pública;
6. integrar ética, accesibilidad, privacidad, evidencia y gobernanza desde el diseño;
7. avanzar hacia un horizonte Full N4 sin saltar controles críticos.

## 4. Principio rector

La fórmula heredada de Viernes de IA se conserva y se amplía:

```text
Pregunta educativa → práctica situada → producto → evidencia → reflexión ética → documentación activa → archivo → transferencia → continuidad
```

Esta fórmula guía toda la estructura del proyecto.

---

## 5. Arquitectura técnica-documental

```text
Local / VS Code
  → producción, edición, revisión y control inicial

GitHub
  → repositorio oficial, control de versiones, issues, releases y manifest

Obsidian
  → segundo cerebro, mapas conceptuales, enlaces, navegación interpretativa

WordPress
  → capa pública futura para recursos curados, publicaciones y comunicación

Exportaciones
  → Markdown, JSON, HTML, PDF, CSV u otros formatos según necesidad
```

Regla de oro:

```text
GitHub conserva la fuente versionada.
Obsidian interpreta y conecta.
WordPress comunica.
Local / VS Code produce.
```

## 8. Fases de implementación

### Fase 0 — Activación documental inmediata

**Objetivo:** crear infraestructura mínima para que el proyecto exista como repositorio vivo.

**Entregables:**

- repositorio local;
- repositorio GitHub;
- README;
- manifest;
- bitácora;
- registro de decisiones;
- changelog;
- primer de continuidad;
- plan maestro.

### Fase 1 — Fundamentos teóricos

**Objetivo:** construir el fundamento conceptual antes de métodos y plantillas.

**Entregables:**

- matriz de fuentes;
- fundamentos teóricos;
- glosario crítico;
- mapa conceptual;
- principios rectores.

### Fase 2 — Arquitectura evolutiva

**Objetivo:** explicar cómo el calendario se convierte en ecosistema.

**Entregables:**

- documento puente;
- marco de proyectos transversales;
- modelo de formación permanente;
- modelo de comunidad de práctica.

### Fase 3 — Gobernanza ética y evidencia

**Objetivo:** establecer reglas de cuidado, consentimiento, privacidad, accesibilidad y uso de evidencia.

**Entregables:**

- protocolo de evidencia;
- consentimiento;
- política de privacidad;
- matriz de riesgos;
- checklist de accesibilidad;
- esquema JSON de evidence ledger.

### Fase 4 — Operación formativa

**Objetivo:** convertir fundamentos y metodología en rutas de estudio, retos, módulos y acompañamiento.

**Entregables:**

- guías de sesión;
- plantillas;
- rutas de estudio independiente;
- módulos;
- secuencias formativas.

### Fase 5 — Transferencia y escalamiento

**Objetivo:** publicar, transferir, auditar y evolucionar hacia N4.

**Entregables:**

- WordPress curado;
- podcast;
- MOOC o certificación;
- informes;
- comunidad de práctica;
- auditoría ligera N4.

## 10. Línea futura: EGIA Quest

Se identifica como línea futura una aplicación educativa lúdica llamada tentativamente **EGIA Quest**.

Propósito:

- motivar práctica continua;
- resolver dilemas éticos;
- practicar buenas prácticas con IA;
- desarrollar competencias múltiples;
- avanzar por niveles;
- otorgar puntos y badges;
- documentar mini-bitácoras;
- exportar perfil y progreso en JSON.

Reglas:

- no infantilizar la experiencia;
- no promover competencia tóxica;
- no premiar velocidad por encima de reflexión;
- no publicar evidencias sin autorización;
- permitir alias y privacidad por defecto;
- permitir exportación portable de perfil/progreso en JSON.

Ubicación sugerida del prompt MVP:


## 13. Cierre ejecutivo

El proyecto **Educación General e Inteligencia Artificial en Acción** entra en una fase decisiva: dejar de ser una extensión de un calendario y comenzar a operar como ecosistema documentado.

La próxima acción no debe ser producir formularios ni protocolos todavía. Debe ser crear fundamentos teóricos con trazabilidad de fuentes, porque el N4 no se sostiene con entusiasmo solamente: necesita raíz, archivo, evidencia y gobernanza.
