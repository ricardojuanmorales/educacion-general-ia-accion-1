# Checklist de Accesibilidad DUA / POUR v0.1
## EGIA-Accion

## Criterios mínimos

- [ ] Lenguaje claro.
- [ ] Estructura navegable.
- [ ] Encabezados correctos.
- [ ] Formatos editables.
- [ ] Texto alternativo en imágenes.
- [ ] Subtítulos o transcripción para audio/video.
- [ ] Contraste visual suficiente.
- [ ] Enlaces descriptivos.
- [ ] Múltiples formas de representación.
- [ ] Múltiples formas de participación.
- [ ] Múltiples formas de expresión.
- [ ] Compatibilidad razonable con lectores de pantalla.

## Plataformas

| Plataforma | Revisión |
|---|---|
| Markdown | Encabezados, enlaces, listas, tablas simples |
| JSON | Campos claros, estructura estable, documentación de esquema |
| GitHub | README y navegación por carpetas |
| Obsidian | Enlaces legibles y etiquetas consistentes |
| WordPress | Alt text, headings, contraste, lectura clara |
