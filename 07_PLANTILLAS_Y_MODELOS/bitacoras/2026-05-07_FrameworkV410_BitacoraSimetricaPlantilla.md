## Propósito de esta bitácora

Esta bitácora acompaña, documenta y ordena la activación del **Framework Genérico V4.1.0** en un proyecto concreto. Funciona como una guía de trabajo humano-IA para transformar una necesidad inicial en un proceso trazable, ético, documentado, evaluable y retomable.

## Principio de genericidad y adopción de estilo

Esta bitácora es **genérica por diseño**. No impone un estilo único, una disciplina específica ni una personalidad fija. Al activarse para un proyecto particular, debe adoptar progresivamente:

- el lenguaje del usuario;
- el nivel de formalidad requerido;
- el campo disciplinar o transdisciplinario del proyecto;
- las convenciones institucionales, comunitarias, académicas, artísticas o técnicas pertinentes;
- los nombres de carpetas, documentos, versiones y productos usados por la persona o equipo;
- el tono de trabajo que mejor sirva al contexto: exploratorio, formal, pedagógico, técnico, creativo, comunitario, institucional o híbrido.

Si el usuario no define estilo, la bitácora operará con un tono claro, sobrio, documentable y flexible. La voz del proyecto debe emerger del uso, no de una plantilla rígida.

## Activación automática de bitácora de sesión

Cada vez que esta bitácora sea invocada para iniciar, continuar o revisar un proyecto, debe activarse de inmediato una **Bitácora de Sesión**. Esta bitácora de sesión registra el estado vivo de la interacción y evita que el trabajo quede flotando sin ancla documental.

Al invocarse, crear o actualizar esta entrada mínima:

```text
# Bitácora de Sesión

Fecha:
Hora:
Proyecto:
Persona o equipo responsable:
Motivo de activación:
Estado previo del proyecto:
Nivel N1-N4 vigente o preliminar:
Perfil / Caleidoscopio vigente o preliminar:
Objetivo de esta sesión:
Documentos abiertos o relevantes:
Decisiones previas que deben respetarse:
Riesgos o alertas activas:
Primer próximo paso:
```

La Bitácora de Sesión debe actualizarse durante la conversación cuando ocurra cualquiera de estos eventos:

- se tome una decisión;
- cambie el nivel N1-N4;
- cambie el perfil o modo Caleidoscopio;
- se produzca una evidencia;
- se identifique un riesgo;
- se aplique un Gate o CHECK;
- se defina una próxima acción;
- se cierre la sesión.

Esta regla convierte cada conversación en una estación de trabajo documentada.

La lógica de esta bitácora es simétrica al framework:

1. Primero identifica la necesidad.
2. Luego determina el nivel N1-N4.
3. Después activa el perfil transversal o modo Caleidoscopio.
4. Construye el brief.
5. Diseña la ruta de trabajo.
6. Produce evidencias.
7. Aplica Gates y CHECKS.
8. Cierra sesión.
9. Deja continuidad.
10. Archiva o prepara transferencia.

> Antes de correr, el proyecto aprende a dejar huellas.

## 8.2 Estructura completa de repositorio o cartapacio

Crear siempre la estructura completa de cartapacios desde el inicio, aunque algunos permanezcan vacíos al comienzo. Esto evita reconstrucciones posteriores y permite que el proyecto crezca sin perder orientación. El nivel N1-N4 determina cuánto se documenta dentro de cada cartapacio, no si el cartapacio existe.

```text
20_MANTENIMIENTO_Y_EVOLUCION/
21_WIKI_DOCUMENTACION_HUMANA/
99_ARCHIVO_HISTORICO/
```

### Regla operativa

- La estructura completa se crea siempre.
- Los cartapacios no utilizados se mantienen vacíos o con un archivo marcador.
- Cada cartapacio puede incluir un `README.md` breve cuando el nivel del proyecto lo requiera.
- El proyecto puede añadir subcartapacios, pero no debe romper la estructura base.
- El cartapacio `00_CONTROL_MAESTRO/` contiene la bitácora principal, bitácoras de sesión, brief, changelog, registro de decisiones y primer de continuidad.
- El cartapacio `99_ARCHIVO_HISTORICO/` conserva versiones cerradas, reemplazadas u obsoletas.
